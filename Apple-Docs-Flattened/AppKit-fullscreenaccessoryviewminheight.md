# fullScreenAccessoryViewMinHeight
The minimum height of the toolbar’s full screen accessory view.
macOS 10.7–10.13Deprecated
```

```

Deprecated
Use an [`NSTitlebarAccessoryViewController`](https://developer.apple.com/documentation/appkit/nstitlebaraccessoryviewcontroller) and its [`fullScreenMinHeight`](https://developer.apple.com/documentation/appkit/nstitlebaraccessoryviewcontroller/fullscreenminheight) property with an [`NSWindow`](https://developer.apple.com/documentation/appkit/nswindow) instead.
## [Discussion](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewminheight#Discussion)
The [`fullScreenAccessoryViewMinHeight`](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewminheight) is used when the menu bar is hidden. During the reveal, the toolbar’s accessory view’s frame is interpolated between its [`fullScreenAccessoryViewMinHeight`](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewminheight) and [`fullScreenAccessoryViewMaxHeight`](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewmaxheight).
If the minimum height is 0 the accessory view isn’t resized; instead a special transition is used to reveal it with the menu bar. This simplifies the accessory view’s task, because it doesn’t have to handle the case of the height being set to `0`.
To create a fixed-height accessory view, set the [`fullScreenAccessoryViewMaxHeight`](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewmaxheight) and [`fullScreenAccessoryViewMinHeight`](https://developer.apple.com/documentation/appkit/nstoolbar/fullscreenaccessoryviewminheight) properties to be equal.
The default value is 0.
