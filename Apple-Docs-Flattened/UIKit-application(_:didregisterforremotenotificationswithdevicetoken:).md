# application(_:didRegisterForRemoteNotificationsWithDeviceToken:)
Tells the delegate that the app successfully registered with Apple Push Notification service (APNs).
iOS 3.0+iPadOS 3.0+Mac Catalyst 13.1+tvOSvisionOS 1.0+
```
@[MainActor](https://developer.apple.com/documentation/Swift/MainActor)
optional func application(
    _ application: [UIApplication](https://developer.apple.com/documentation/uikit/uiapplication),
    didRegisterForRemoteNotificationsWithDeviceToken deviceToken: [Data](https://developer.apple.com/documentation/Foundation/Data)
)
```

## [ Parameters ](https://developer.apple.com/documentation/uikit/uiapplicationdelegate/application\(_:didregisterforremotenotificationswithdevicetoken:\)#parameters) 

`application`
    
The app object that initiated the remote-notification registration process. 

`deviceToken`
    
A globally unique token that identifies this device to APNs. Send this token to the server that you use to generate remote notifications. Your server must pass this token unmodified back to APNs when sending those remote notifications.
APNs device tokens are of variable length. Do not hard-code their size.
## [Discussion](https://developer.apple.com/documentation/uikit/uiapplicationdelegate/application\(_:didregisterforremotenotificationswithdevicetoken:\)#Discussion)
UIKit calls this method after it successfully registers your app with APNs. In your implementation of this method, send the contents of the `deviceToken` parameter to the server that you use to generate remote notifications. Never cache the device token locally on the user’s device. Device tokens can change periodically, so caching the value risks sending an invalid token to your server. If the device token hasn’t changed, registering with APNs and returning the token happens quickly.
Typically, this method is called only after you call the [`registerForRemoteNotifications()`](https://developer.apple.com/documentation/uikit/uiapplication/registerforremotenotifications\(\)) method of [`UIApplication`](https://developer.apple.com/documentation/uikit/uiapplication), but UIKit might call it in other rare circumstances. For example, UIKit calls the method when the user launches an app after having restored a device from data that is not the device’s backup data. In this exceptional case, the app won’t know the new device’s token until the user launches it.
