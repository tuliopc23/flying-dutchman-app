# init(sidebarWithViewController:)
Creates a split view item that represents a sidebar for the specified view controller.
macOS 10.11+
```
convenience init(sidebarWithViewController viewController: [NSViewController](https://developer.apple.com/documentation/appkit/nsviewcontroller))
```

## [Discussion](https://developer.apple.com/documentation/appkit/nssplitviewitem/init\(sidebarwithviewcontroller:\)#Discussion)
Sidebar items take advantage of the standard system appearance and behavior for sidebars, including:
  * The translucent material background
  * The ability to collapse and expand on split view size changes
  * The ability to overlay at small split view sizes in full-screen mode


Additionally, sidebars use standard system default values for these properties:
  * [`canCollapse`](https://developer.apple.com/documentation/appkit/nssplitviewitem/cancollapse) and [`isSpringLoaded`](https://developer.apple.com/documentation/appkit/nssplitviewitem/isspringloaded) are [`true`](https://developer.apple.com/documentation/Swift/true)
  * [`minimumThickness`](https://developer.apple.com/documentation/appkit/nssplitviewitem/minimumthickness) and [`maximumThickness`](https://developer.apple.com/documentation/appkit/nssplitviewitem/maximumthickness) use the standard minimum and maximum sidebar size
  * [`preferredThicknessFraction`](https://developer.apple.com/documentation/appkit/nssplitviewitem/preferredthicknessfraction) uses the standard fraction for sidebars (`0.15`)
