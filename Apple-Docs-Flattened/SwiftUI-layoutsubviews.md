# LayoutSubviews
A collection of proxy values that represent the subviews of a layout view.
iOS 16.0+iPadOS 16.0+Mac Catalyst 16.0+macOS 13.0+tvOS 16.0+visionOS 1.0+watchOS 9.0+
```
struct LayoutSubviews
```

## [Overview](https://developer.apple.com/documentation/swiftui/layoutsubviews#overview)
You receive a `LayoutSubviews` input to your implementations of [`Layout`](https://developer.apple.com/documentation/swiftui/layout) protocol methods, like [`placeSubviews(in:proposal:subviews:cache:)`](https://developer.apple.com/documentation/swiftui/layout/placesubviews\(in:proposal:subviews:cache:\)) and [`sizeThatFits(proposal:subviews:cache:)`](https://developer.apple.com/documentation/swiftui/layout/sizethatfits\(proposal:subviews:cache:\)). The `subviews` parameter (which the protocol aliases to the [`Layout.Subviews`](https://developer.apple.com/documentation/swiftui/layout/subviews) type) is a collection that contains proxies for the layout’s subviews (of type [`LayoutSubview`](https://developer.apple.com/documentation/swiftui/layoutsubview)). The proxies appear in the collection in the same order that they appear in the [`ViewBuilder`](https://developer.apple.com/documentation/swiftui/viewbuilder) input to the layout container. Use the proxies to perform layout operations.
Access the proxies in the collection as you would the contents of any Swift random-access collection. For example, you can enumerate all of the subviews and their indices to inspect or operate on them:
```
for (index, subview) in subviews.enumerated() {
    // ...
}

```
