# ARGeoTrackingStatus.StateReason.geoDataNotLoaded
A state in which the framework downloads localization imagery.
iOS 14.0+iPadOS 14.0+Mac Catalyst 14.0+
```
case geoDataNotLoaded
```

## [Discussion](https://developer.apple.com/documentation/arkit/argeotrackingstatus/statereason-swift.enum/geodatanotloaded#Discussion)
ARKit provides this reason in state [`ARGeoTrackingStatus.State.localizing`](https://developer.apple.com/documentation/arkit/argeotrackingstatus/state-swift.enum/localizing) when the session is actively attempting to download localization imagery (see [Refine the user’s position with imagery](https://developer.apple.com/documentation/arkit/argeotrackingconfiguration#Refine-the-users-position-with-imagery)).
If this state persists for too long, it may indicate a network issue. If a reasonable amount of time elapses in this state reason, the app may consider requesting that the user check their internet connection.
