# AVCaptureTimecode.SourceType
Defines possible sources for generating timecode in using a timecode generator.
iOS 26.0+iPadOS 26.0+Mac Catalyst 26.0+macOS 26.0+tvOS 26.0+
```
enum SourceType
```

## [Topics](https://developer.apple.com/documentation/avfoundation/avcapturetimecode/sourcetype-swift.enum#topics)
### [Source types](https://developer.apple.com/documentation/avfoundation/avcapturetimecode/sourcetype-swift.enum#Source-types)
[`case external`](https://developer.apple.com/documentation/avfoundation/avcapturetimecode/sourcetype-swift.enum/external)
Synchronizes timecode to an external timecode data stream. Ideal for professional audio and video synchronization with external quarter-frame MIDI or HID timecode hardware.
[`case frameCount`](https://developer.apple.com/documentation/avfoundation/avcapturetimecode/sourcetype-swift.enum/framecount)
No internal or external source is adopted. Timecodes are zero-based, sequentially generated frame counts.
[`case realTimeClock`](https://developer.apple.com/documentation/avfoundation/avcapturetimecode/sourcetype-swift.enum/realtimeclock)
Synchronizes timecode to the system clock for real-time applications. Useful for live events or scenarios requiring alignment with the actual time of day.
