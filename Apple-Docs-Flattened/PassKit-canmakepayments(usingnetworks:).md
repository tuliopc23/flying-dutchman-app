# canMakePayments(usingNetworks:)
Returns whether the user can make payments through the specified network.
iOS 10.0+iPadOS 10.0+Mac Catalyst 13.1+macOS 11.0+visionOS 1.0+watchOS 3.0+
```
class func canMakePayments(usingNetworks supportedNetworks: [[PKPaymentNetwork](https://developer.apple.com/documentation/passkit/pkpaymentnetwork)]) -> [Bool](https://developer.apple.com/documentation/Swift/Bool)
```

## [ Parameters ](https://developer.apple.com/documentation/passkit/pkpaymentauthorizationcontroller/canmakepayments\(usingnetworks:\)#parameters) 

`supportedNetworks`
    
An array of payment networks, as listed in Payment Networks.
## [Return Value](https://developer.apple.com/documentation/passkit/pkpaymentauthorizationcontroller/canmakepayments\(usingnetworks:\)#return-value)
[`true`](https://developer.apple.com/documentation/Swift/true) if the user can make payments through any of the specified networks; otherwise, [`false`](https://developer.apple.com/documentation/Swift/false).
## [Discussion](https://developer.apple.com/documentation/passkit/pkpaymentauthorizationcontroller/canmakepayments\(usingnetworks:\)#Discussion)
User may not be able to make payments for a variety of reasons. For example, this functionality may not be supported by their hardware, or it may be restricted by parental controls.
If there are no configured payment cards, this method always returns [`false`](https://developer.apple.com/documentation/Swift/false).
