# touchesMoved(_:with:)
Sent to the gesture recognizer when one or more fingers move in the associated view.
iOSiPadOSMac CatalysttvOSvisionOS

```
    _ touches: [Set](https://developer.apple.com/documentation/Swift/Set)<[UITouch](https://developer.apple.com/documentation/uikit/uitouch)>,
    with event: [UIEvent](https://developer.apple.com/documentation/uikit/uievent)
)
```

## [ Parameters ](https://developer.apple.com/documentation/uikit/uigesturerecognizer/touchesmoved\(_:with:\)#parameters) 

`touches`
    
A set of [`UITouch`](https://developer.apple.com/documentation/uikit/uitouch) instances in the event represented by `event` that represent touches in the [`UITouch.Phase.moved`](https://developer.apple.com/documentation/uikit/uitouch/phase-swift.enum/moved) phase. 

`event`
    
A [`UIEvent`](https://developer.apple.com/documentation/uikit/uievent) object representing the event to which the touches belong.
## [Discussion](https://developer.apple.com/documentation/uikit/uigesturerecognizer/touchesmoved\(_:with:\)#Discussion)
This method has the same exact signature as the corresponding one declared by [`UIResponder`](https://developer.apple.com/documentation/uikit/uiresponder). Through this method a gesture recognizer receives touch objects (in their [`UITouch.Phase.moved`](https://developer.apple.com/documentation/uikit/uitouch/phase-swift.enum/moved) phase) before the view attached to the gesture recognizer receives them. `UIGestureRecognizer` objects are not in the responder chain, yet observe touches hit-tested to their view and their view’s subviews. After observation, the delivery of touch objects to the attached view, or their disposition otherwise, is affected by the [`cancelsTouchesInView`](https://developer.apple.com/documentation/uikit/uigesturerecognizer/cancelstouchesinview), [`delaysTouchesBegan`](https://developer.apple.com/documentation/uikit/uigesturerecognizer/delaystouchesbegan), and [`delaysTouchesEnded`](https://developer.apple.com/documentation/uikit/uigesturerecognizer/delaystouchesended) properties.
If the gesture recognizer is interpreting a continuous gesture, it should set its state to [`UIGestureRecognizer.State.changed`](https://developer.apple.com/documentation/uikit/uigesturerecognizer/state-swift.enum/changed) upon receiving this message. If at any point in its handling of the touch objects the gesture recognizer determines that the multi-touch event sequence is not its gesture, it should set it state to [`UIGestureRecognizer.State.cancelled`](https://developer.apple.com/documentation/uikit/uigesturerecognizer/state-swift.enum/cancelled) .
Multiple touches are disabled by default. In order to receive multiple touch events you must set the a [`isMultipleTouchEnabled`](https://developer.apple.com/documentation/uikit/uiview/ismultipletouchenabled) property of the attached view instance to [`true`](https://developer.apple.com/documentation/Swift/true).
