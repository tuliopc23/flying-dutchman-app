# init(contentsOfURL:encoding:)
Returns an `NSString` object initialized by reading data from a given URL interpreted using a given encoding.
iOS 2.0+iPadOS 2.0+Mac Catalyst 13.0+macOS 10.0+tvOS 9.0+visionOS 1.0+watchOS 2.0+
```
convenience init(
    contentsOfURL url: [URL](https://developer.apple.com/documentation/foundation/url),
    encoding enc: [UInt](https://developer.apple.com/documentation/Swift/UInt)
) throws
```

convenience init(

) throws

## [ Parameters ](https://developer.apple.com/documentation/Foundation/NSString/init\(contentsOfURL:encoding:\)-715fw#parameters) 

`url`
    
The URL to read. 

`enc`
    
The encoding of the file at `path`. For possible values, see [`NSStringEncoding`](https://developer.apple.com/documentation/foundation/nsstringencoding).
## [Return Value](https://developer.apple.com/documentation/Foundation/NSString/init\(contentsOfURL:encoding:\)-715fw#return-value)
An `NSString` object initialized by reading data from `url`. The returned object may be different from the original receiver. If the URL can’t be opened or there is an encoding error, returns `nil`.
## [Discussion](https://developer.apple.com/documentation/Foundation/NSString/init\(contentsOfURL:encoding:\)-715fw#Discussion)
Handling Errors in Swift
In Swift, this method returns a nonoptional result and is marked with the `throws` keyword to indicate that it throws an error in cases of failure.
You call this method in a `try` expression and handle any errors in the `catch` clauses of a `do` statement, as described in [Error Handling](https://docs.swift.org/swift-book/LanguageGuide/ErrorHandling.html) in [The Swift Programming Language](https://docs.swift.org/swift-book/) and `About Imported Cocoa Error Parameters`.
