# voiceChat
A mode that indicates that your app is performing two-way voice communication, such as using Voice over Internet Protocol (VoIP).
iOS 5.0+iPadOS 5.0+Mac Catalyst 13.1+tvOS 9.0+visionOS 1.0+watchOS 2.0+
```
static let voiceChat: [AVAudioSession](https://developer.apple.com/documentation/avfaudio/avaudiosession).[Mode](https://developer.apple.com/documentation/avfaudio/avaudiosession/mode-swift.struct)
```

## [Discussion](https://developer.apple.com/documentation/AVFAudio/AVAudioSession/Mode-swift.struct/voiceChat#discussion)
This mode is appropriate for Voice over IP (VoIP) apps that use the [`playAndRecord`](https://developer.apple.com/documentation/avfaudio/avaudiosession/category-swift.struct/playandrecord) category. When you set this mode, the session optimizes the device’s tonal equalization for voice and reduces the set of allowed audio routes to only those suitable for voice chat. Setting this mode also causes the system to automatically apply the [`allowBluetoothHFP`](https://developer.apple.com/documentation/avfaudio/avaudiosession/categoryoptions-swift.struct/allowbluetoothhfp) category option.
VoIP apps should also use the [Audio Unit Voice I/O](https://developer.apple.com/documentation/AudioToolbox/audio-unit-voice-i-o) as it provides useful voice features, including automatic gain correction, adjustment of voice processing, and muting. When an app uses this Audio Unit without explicitly setting its mode to a chat variant (voice, video, or game), the session sets the [`voiceChat`](https://developer.apple.com/documentation/avfaudio/avaudiosession/mode-swift.struct/voicechat) mode implicitly. However, if an app previously set its category to [`playAndRecord`](https://developer.apple.com/documentation/avfaudio/avaudiosession/category-swift.struct/playandrecord) and mode to [`videoChat`](https://developer.apple.com/documentation/avfaudio/avaudiosession/mode-swift.struct/videochat) or [`gameChat`](https://developer.apple.com/documentation/avfaudio/avaudiosession/mode-swift.struct/gamechat), instantiating the Audio Unit doesn’t cause the mode to change.
For apps that use one or more chat modes (voice, video, or game), but don’t use [Audio Unit Voice I/O](https://developer.apple.com/documentation/AudioToolbox/audio-unit-voice-i-o) or [`AVAudioEngine`](https://developer.apple.com/documentation/avfaudio/avaudioengine) with [`setVoiceProcessingEnabled(_:)`](https://developer.apple.com/documentation/avfaudio/avaudioionode/setvoiceprocessingenabled\(_:\)), the system reduces the processing it applies to audio signals. Specifically, it doesn’t apply voice-specific processing, like echo cancellation and automatic gain correction, and disables dynamic processing on input and output, which results in a lower playback level.
