# EditButton
A button that toggles the edit mode environment value.
iOS 13.0+iPadOS 13.0+Mac Catalyst 13.0+visionOS 1.0+
```
struct EditButton
```

## [Overview](https://developer.apple.com/documentation/swiftui/editbutton#overview)
An edit button toggles the environment’s [`editMode`](https://developer.apple.com/documentation/swiftui/environmentvalues/editmode) value for content within a container that supports edit mode. In the following example, an edit button placed inside a [`NavigationView`](https://developer.apple.com/documentation/swiftui/navigationview) supports editing of a [`List`](https://developer.apple.com/documentation/swiftui/list):
```
@State private var fruits = [
    "Apple",
    "Banana",
    "Papaya",
    "Mango"
]


var body: some View {
    NavigationView {
        List {
            ForEach(fruits, id: \.self) { fruit in
                Text(fruit)
            }
            .onDelete { fruits.remove(atOffsets: $0) }
            .onMove { fruits.move(fromOffsets: $0, toOffset: $1) }
        }
        .navigationTitle("Fruits")
        .toolbar {
            EditButton()
        }
    }
}

```

Because the [`ForEach`](https://developer.apple.com/documentation/swiftui/foreach) in the above example defines behaviors for [`onDelete(perform:)`](https://developer.apple.com/documentation/swiftui/dynamicviewcontent/ondelete\(perform:\)) and [`onMove(perform:)`](https://developer.apple.com/documentation/swiftui/dynamicviewcontent/onmove\(perform:\)), the editable list displays the delete and move UI when the user taps Edit. Notice that the Edit button displays the title “Done” while edit mode is active:
![A screenshot of an app with an Edit button in the navigation bar.](https://docs-assets.developer.apple.com/published/ef01598c8da781c8c60c9e3bcaa97418/EditButton-1%402x.png)
You can also create custom views that react to changes in the edit mode state, as described in [`EditMode`](https://developer.apple.com/documentation/swiftui/editmode).
