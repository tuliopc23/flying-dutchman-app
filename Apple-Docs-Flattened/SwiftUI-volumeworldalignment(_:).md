# volumeWorldAlignment(_:)
Specifies how a volume should be aligned when moved in the world.
visionOS 2.0+
```
nonisolated
func volumeWorldAlignment(_ behavior: [WorldAlignmentBehavior](https://developer.apple.com/documentation/swiftui/worldalignmentbehavior)) -> some [Scene](https://developer.apple.com/documentation/swiftui/scene)

```

## [Discussion](https://developer.apple.com/documentation/swiftui/scene/volumeworldalignment\(_:\)#discussion)
For example, you can create a volume that remains parallel to the floor even when lifted up high above eye level by applying a [`gravityAligned`](https://developer.apple.com/documentation/swiftui/worldalignmentbehavior/gravityaligned) alignment to the scene:
```
@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
        .windowStyle(.volumetric)
        .volumeWorldAlignment(.gravityAligned)
    }
}

```

The default value if you don’t apply the modifier is [`automatic`](https://developer.apple.com/documentation/swiftui/worldalignmentbehavior/automatic). With that strategy, volumes will tilt themselves so the front remains fully visible while being repositioned.
