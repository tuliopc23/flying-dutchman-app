# map(_:)
Transforms all elements from the upstream publisher with a provided closure.

## [Discussion](https://developer.apple.com/documentation/Combine/Publisher/map\(_:\)-99evh#discussion)
Combine’s [`map(_:)`](https://developer.apple.com/documentation/combine/publisher/map\(_:\)-99evh) operator performs a function similar to that of [`map(_:)`](https://developer.apple.com/documentation/Swift/Sequence/map\(_:\)) in the Swift standard library: it uses a closure to transform each element it receives from the upstream publisher. You use [`map(_:)`](https://developer.apple.com/documentation/combine/publisher/map\(_:\)-99evh) to transform from one kind of element to another.
The following example uses an array of numbers as the source for a collection based publisher. A [`map(_:)`](https://developer.apple.com/documentation/combine/publisher/map\(_:\)-99evh) operator consumes each integer from the publisher and uses a dictionary to transform it from its Arabic numeral to a Roman equivalent, as a [`String`](https://developer.apple.com/documentation/Swift/String). If the [`map(_:)`](https://developer.apple.com/documentation/combine/publisher/map\(_:\)-99evh)’s closure fails to look up a Roman numeral, it returns the string `(unknown)`.
```
let numbers = [5, 4, 3, 2, 1, 0]
let romanNumeralDict: [Int : String] =
   [1:"I", 2:"II", 3:"III", 4:"IV", 5:"V"]
cancellable = numbers.publisher
    .map { romanNumeralDict[$0] ?? "(unknown)" }
    .sink { print("\($0)", terminator: " ") }


// Prints: "V IV III II I (unknown)"

```

If your closure can throw an error, use Combine’s [`tryMap(_:)`](https://developer.apple.com/documentation/combine/publisher/trymap\(_:\)) operator instead.
