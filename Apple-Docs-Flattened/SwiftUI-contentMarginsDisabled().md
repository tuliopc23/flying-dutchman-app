# contentMarginsDisabled()
Disable default content margins.
iOS 15.0+iPadOS 15.0+Mac Catalyst 15.0+macOS 12.0+visionOS 26.0+watchOS 9.0+
```
@[MainActor](https://developer.apple.com/documentation/Swift/MainActor) @preconcurrency
func contentMarginsDisabled() -> some [WidgetConfiguration](https://developer.apple.com/documentation/swiftui/widgetconfiguration)

```

## [Return Value](https://developer.apple.com/documentation/SwiftUI/WidgetConfiguration/contentMarginsDisabled\(\)#return-value)
A modified widget configuration that doesn’t use default content margins.
## [Discussion](https://developer.apple.com/documentation/SwiftUI/WidgetConfiguration/contentMarginsDisabled\(\)#discussion)
When you disable content margins for a widget, the system doesn’t automatically add margins around the widget’s content, and you are responsible for specifying margins and padding around your widget content for each context. To specify custom margins, use [`widgetContentMargins`](https://developer.apple.com/documentation/SwiftUI/EnvironmentValues/widgetContentMargins) in combination with [`padding(_:)`](https://developer.apple.com/documentation/SwiftUI/View/padding\(_:\)) to selectively or partially apply the default content margins.
This modifier has no effect on operation system versions prior to iOS 17, watchOS 10, or macOS 14.
