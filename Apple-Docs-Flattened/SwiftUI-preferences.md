# Preferences
Indicate configuration preferences from views to their container views.
## [Overview](https://developer.apple.com/documentation/swiftui/preferences#Overview)
Whereas you use the environment to configure the subviews of a view, you use preferences to send configuration information from subviews toward their container. However, unlike configuration information that flows down a view hierarchy from one container to many subviews, a single container needs to reconcile potentially conflicting preferences flowing up from its many subviews.
![](https://docs-assets.developer.apple.com/published/868be1f862daf281c185c168365df04f/preferences-hero%402x.png)
When you use the [`PreferenceKey`](https://developer.apple.com/documentation/swiftui/preferencekey) protocol to define a custom preference, you indicate how to merge preferences from multiple subviews. You can then set a value for the preference on a view using the [`preference(key:value:)`](https://developer.apple.com/documentation/swiftui/view/preference\(key:value:\)) view modifier. Many built-in modifiers, like [`navigationTitle(_:)`](https://developer.apple.com/documentation/swiftui/view/navigationtitle\(_:\)), rely on preferences to send configuration information to their container.
