# init(preferredCompactColumn:sidebar:detail:)
Creates a two-column navigation split view that enables programmatic control over which column appears on top when the view collapses into a single column in narrow sizes.
iOS 17.0+iPadOS 17.0+Mac Catalyst 17.0+macOS 14.0+tvOS 17.0+visionOS 1.0+watchOS 10.0+
```
nonisolated
init(
    preferredCompactColumn: [Binding](https://developer.apple.com/documentation/swiftui/binding)<[NavigationSplitViewColumn](https://developer.apple.com/documentation/swiftui/navigationsplitviewcolumn)>,
    @[ViewBuilder](https://developer.apple.com/documentation/swiftui/viewbuilder) sidebar: () -> Sidebar,
    @[ViewBuilder](https://developer.apple.com/documentation/swiftui/viewbuilder) detail: () -> Detail
) where Content == [EmptyView](https://developer.apple.com/documentation/swiftui/emptyview)
```

Available when `Sidebar` conforms to `View`, `Content` conforms to `View`, and `Detail` conforms to `View`.
## [ Parameters ](https://developer.apple.com/documentation/swiftui/navigationsplitview/init\(preferredcompactcolumn:sidebar:detail:\)#parameters) 

`preferredCompactColumn`
    
A [`Binding`](https://developer.apple.com/documentation/swiftui/binding) to state that controls which column appears on top when the view collapses. 

`sidebar`
    
The view to show in the leading column. 

`detail`
    
The view to show in the detail area.
