# measurement(width:usage:hidesScaleName:numberFormatStyle:)
Returns a format style to format temperature units.
iOS 15.0+iPadOS 15.0+Mac Catalyst 15.0+macOS 12.0+tvOS 15.0+visionOS 1.0+watchOS 8.0+
```
static func measurement(
    width: [Measurement](https://developer.apple.com/documentation/foundation/measurement)<[UnitTemperature](https://developer.apple.com/documentation/foundation/unittemperature)>.[FormatStyle](https://developer.apple.com/documentation/foundation/measurement/formatstyle).[UnitWidth](https://developer.apple.com/documentation/foundation/measurement/formatstyle/unitwidth) = .abbreviated,
    usage: [MeasurementFormatUnitUsage](https://developer.apple.com/documentation/foundation/measurementformatunitusage)<[UnitTemperature](https://developer.apple.com/documentation/foundation/unittemperature)> = .general,
    hidesScaleName: [Bool](https://developer.apple.com/documentation/Swift/Bool) = false,
    numberFormatStyle: [FloatingPointFormatStyle](https://developer.apple.com/documentation/foundation/floatingpointformatstyle)<[Double](https://developer.apple.com/documentation/Swift/Double)>? = nil
) -> Self where Self == [Measurement](https://developer.apple.com/documentation/foundation/measurement)<[UnitTemperature](https://developer.apple.com/documentation/foundation/unittemperature)>.[FormatStyle](https://developer.apple.com/documentation/foundation/measurement/formatstyle)
```

## [ Parameters ](https://developer.apple.com/documentation/Foundation/FormatStyle/measurement\(width:usage:hidesScaleName:numberFormatStyle:\)#parameters) 

`width`
    
The width — such as full names or abbreviations — with which to present units. 

`usage`
    
The contextual usage of the measurement unit, such as whether a temperature applies to a person or to the weather. 

`hidesScaleName`
    
A Boolean value that directs the formatter to hide the name of the scale (Kelvin, degrees Celsius, or degrees Fahrenheit). Defaults to `false`. 

`numberFormatStyle`
    
The format style with which to format the numeric part of the temperature.
## [Return Value](https://developer.apple.com/documentation/Foundation/FormatStyle/measurement\(width:usage:hidesScaleName:numberFormatStyle:\)#return-value)
A format style that formats measurements according to the given parameters.
## [Discussion](https://developer.apple.com/documentation/Foundation/FormatStyle/measurement\(width:usage:hidesScaleName:numberFormatStyle:\)#Discussion)
Use this static method when the call point allows the use of [`Measurement.FormatStyle`](https://developer.apple.com/documentation/foundation/measurement/formatstyle) and the value type is `Measurement<UnitTemperature>`. You typically do this when calling the [`formatted(_:)`](https://developer.apple.com/documentation/foundation/measurement/formatted\(_:\)) method of [`Measurement`](https://developer.apple.com/documentation/foundation/measurement).
The following example creates an array of [`Measurement`](https://developer.apple.com/documentation/foundation/measurement) values that represent body temperatures from a human patient. It then uses [`formatted(_:)`](https://developer.apple.com/documentation/foundation/measurement/formatted\(_:\)) and the format style provided by this method to format the distances. The style specifies the [`person`](https://developer.apple.com/documentation/foundation/measurementformatunitusage/person-4ifk7) usage to clarify that the temperatures refer to a person’s body temperature, as opposed to other uses like weather.
```
let rawTemparatures: [Double] = [36.4, 36.6, 37.0, 36.9, 36.7]
let temperatures = rawTemparatures.map { Measurement(value: $0, unit: UnitTemperature.celsius) }
let formattedTemperatures = temperatures.map { $0.formatted(
    .measurement(width: .abbreviated,
                 usage: .person,
                 hidesScaleName: false,
                 numberFormatStyle: .number)) } // ["36.4°C", "36.6°C", "37°C", "36.9°C", "36.7°C"]

```
