# run(_:)
Run the listener and receive incoming connections.
iOS 26.0+iPadOS 26.0+Mac Catalyst 26.0+macOS 26.0+tvOS 26.0+visionOS 26.0+watchOS 26.0+
```
final func run(_ handler: @escaping @isolated(any) ([NetworkConnection](https://developer.apple.com/documentation/network/networkconnection)<ApplicationProtocol>) async throws -> [Void](https://developer.apple.com/documentation/Swift/Void)) async throws
```

Available when `ApplicationProtocol` conforms to `OneToOneProtocol`.
## [ Parameters ](https://developer.apple.com/documentation/Network/NetworkListener/run\(_:\)-4iov3#parameters) 

`handler`
    
A handler to receive incoming connections.
## [Discussion](https://developer.apple.com/documentation/Network/NetworkListener/run\(_:\)-4iov3#discussion)
When the listener state moves to ready, the listener is registered with the system and can receive incoming connections.
`run()` should only be called once on a listener, and multiple calls to `run()` will throw an exception.
The closure inherits the isolation domain of the caller.
