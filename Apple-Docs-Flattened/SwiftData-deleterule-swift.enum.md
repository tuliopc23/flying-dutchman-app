# Schema.Relationship.DeleteRule
Describes the rule to apply when deleting a model containing references to other models.
iOS 17.0+iPadOS 17.0+Mac Catalyst 17.0+macOS 14.0+tvOS 17.0+visionOS 1.0+watchOS 10.0+Swift 5.9+

### [Accessing delete rules](https://developer.apple.com/documentation/swiftdata/schema/relationship/deleterule-swift.enum#Accessing-delete-rules)
[`case cascade`](https://developer.apple.com/documentation/swiftdata/schema/relationship/deleterule-swift.enum/cascade)
A rule that deletes any related models.
[`case deny`](https://developer.apple.com/documentation/swiftdata/schema/relationship/deleterule-swift.enum/deny)
A rule that prevents the deletion of a model because it contains one or more references to other models.
[`case noAction`](https://developer.apple.com/documentation/swiftdata/schema/relationship/deleterule-swift.enum/noaction)
A rule that doesn’t make changes to any related models.
[`case nullify`](https://developer.apple.com/documentation/swiftdata/schema/relationship/deleterule-swift.enum/nullify)
A rule that nullifies the related model’s reference to the deleted model.
