# maximumNumberOfLines
The maximum number of lines a wrapping text field displays before clipping or truncating the text.
macOS 10.11+
```
@[MainActor](https://developer.apple.com/documentation/Swift/MainActor)
var maximumNumberOfLines: [Int](https://developer.apple.com/documentation/Swift/Int) { get set }
```

## [Discussion](https://developer.apple.com/documentation/appkit/nstextfield/maximumnumberoflines#Discussion)
The default value of `0` indicates no limit to the number of lines, and the text fills the bounds of the text field cell.
If the text field reaches the maximum number of lines, or if the height of the container can’t accommodate the number of lines, the text field clips or truncates the text, depending on the cell’s [`truncatesLastVisibleLine`](https://developer.apple.com/documentation/appkit/nscell/truncateslastvisibleline) setting.
Important
This value also affects [`sizeThatFits(_:)`](https://developer.apple.com/documentation/appkit/nscontrol/sizethatfits\(_:\)), [`fittingSize`](https://developer.apple.com/documentation/appkit/nsview/fittingsize), and [`intrinsicContentSize`](https://developer.apple.com/documentation/appkit/nsview/intrinsiccontentsize). If the value of this property isn’t `1`, the text field may use multiple lines to determine its intrinsic content size.
