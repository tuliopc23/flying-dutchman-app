Instance Method
# getObjectValue(_:for:range:)
Returns by reference a cell-content object after creating it from a range of characters in a given string.
iOS 2.0+iPadOS 2.0+Mac Catalyst 13.0+macOS 10.0+tvOS 9.0+visionOS 1.0+watchOS 2.0+
```
func getObjectValue(
    _ obj: [AutoreleasingUnsafeMutablePointer](https://developer.apple.com/documentation/Swift/AutoreleasingUnsafeMutablePointer)<AnyObject?>?,
    for string: [String](https://developer.apple.com/documentation/Swift/String),
    range rangep: [UnsafeMutablePointer](https://developer.apple.com/documentation/Swift/UnsafeMutablePointer)<NSRange>?
) throws
```

## [ Parameters ](https://developer.apple.com/documentation/foundation/numberformatter/getobjectvalue\(_:for:range:\)#parameters) 

`obj`
    
On return, contains an instance of [`NSDecimalNumber`](https://developer.apple.com/documentation/foundation/nsdecimalnumber) or [`NSNumber`](https://developer.apple.com/documentation/foundation/nsnumber) based on the current value of the [`generatesDecimalNumbers`](https://developer.apple.com/documentation/foundation/numberformatter/generatesdecimalnumbers) property. Returns `nil` by reference if conversion failed. 

`string`
    
A string object with the range of characters specified in `rangep` that is used to create `anObject`. 

`rangep`
    
A range of characters in `aString`. On return, contains the actual range of characters used to create the object.
## [Discussion](https://developer.apple.com/documentation/foundation/numberformatter/getobjectvalue\(_:for:range:\)#Discussion)
If a string contains any characters other than numerical digits or locale-appropriate group or decimal separators, parsing will fail.
Any leading or trailing space separator characters in a string are ignored. For example, the strings “ 5”, “5 “, and “5” all produce the number `5`.
If there is an error, this method calls `control(_:didFailToFormatString:errorDescription:)` on the delegate.
Handling Errors in Swift
In Swift, this method returns `Void` and is marked with the `throws` keyword to indicate that it throws an error in cases of failure.
You call this method in a `try` expression and handle any errors in the `catch` clauses of a `do` statement, as described in [Error Handling](https://docs.swift.org/swift-book/LanguageGuide/ErrorHandling.html) in [The Swift Programming Language](https://docs.swift.org/swift-book/) and `About Imported Cocoa Error Parameters`.
