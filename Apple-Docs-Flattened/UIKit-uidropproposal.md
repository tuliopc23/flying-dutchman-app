# UIDropProposal
A configuration for the behavior of a drop interaction, required if a view accepts drop activities.
iOS 11.0+iPadOS 11.0+Mac Catalyst 13.1+visionOS 1.0+
```
@[MainActor](https://developer.apple.com/documentation/Swift/MainActor)
class UIDropProposal
```

## [ Mentioned in ](https://developer.apple.com/documentation/uikit/uidropproposal#mentions)
[](https://developer.apple.com/documentation/uikit/making-a-view-into-a-drop-destination)
## [Overview](https://developer.apple.com/documentation/uikit/uidropproposal#overview)
If a view’s drop interaction delegate accepts dropped drag items, it must return a drop proposal in its implementation of the [`dropInteraction(_:sessionDidUpdate:)`](https://developer.apple.com/documentation/uikit/uidropinteractiondelegate/dropinteraction\(_:sessiondidupdate:\)) method.
## [Topics](https://developer.apple.com/documentation/uikit/uidropproposal#topics)

Initializes a new drop proposal with a drop operation type.

[`var isPrecise: Bool`](https://developer.apple.com/documentation/uikit/uidropproposal/isprecise)
A Boolean value that proposes that the drop interaction define the drop location precisely, such as at a specific point within existing text.
[`var prefersFullSizePreview: Bool`](https://developer.apple.com/documentation/uikit/uidropproposal/prefersfullsizepreview)
A Boolean value that indicates that the drag item preview should be shown at its full, original size.
