# view(_:didAdd:for:)
Tells the delegate that a SpriteKit node corresponding to a new AR anchor has been added to the scene.
iOS 11.0+iPadOS 11.0+Mac Catalyst 13.1+
```
optional func view(
    _ view: [ARSKView](https://developer.apple.com/documentation/arkit/arskview),
    didAdd node: [SKNode](https://developer.apple.com/documentation/SpriteKit/SKNode),
    for anchor: [ARAnchor](https://developer.apple.com/documentation/arkit/aranchor)
)
```

## [ Parameters ](https://developer.apple.com/documentation/arkit/arskviewdelegate/view\(_:didadd:for:\)#parameters) 

`view`
    
The [`ARSKView`](https://developer.apple.com/documentation/arkit/arskview) object rendering the scene. 

`node`
    
The newly added SpriteKit node. 

`anchor`
    
The AR anchor corresponding to the node.
## [Discussion](https://developer.apple.com/documentation/arkit/arskviewdelegate/view\(_:didadd:for:\)#Discussion)
Depending on the session configuration, ARKit may automatically add anchors to a session. The view calls this method once for each new anchor. ARKit also calls this method to provide visual content for any [`ARAnchor`](https://developer.apple.com/documentation/arkit/aranchor) objects you manually add using the session’s [`add(anchor:)`](https://developer.apple.com/documentation/arkit/arsession/add\(anchor:\)) method.
You can provide visual content for the anchor by adding child nodes.
Alternatively, you can implement the [`view(_:nodeFor:)`](https://developer.apple.com/documentation/arkit/arskviewdelegate/view\(_:nodefor:\)) method to create your own node (or instance of an [`SKNode`](https://developer.apple.com/documentation/SpriteKit/SKNode) subclass) for an anchor.
