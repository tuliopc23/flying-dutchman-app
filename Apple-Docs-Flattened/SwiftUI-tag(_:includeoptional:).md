# tag(_:includeOptional:)
Sets the unique tag value of this view.
iOS 13.0+iPadOS 13.0+Mac Catalyst 13.0+macOS 10.15+tvOS 13.0+visionOS 1.0+watchOS 6.0+
```
nonisolated
func tag<V>(
    _ tag: V,
    includeOptional: [Bool](https://developer.apple.com/documentation/Swift/Bool) = true
) -> some [View](https://developer.apple.com/documentation/swiftui/view) where V : [Hashable](https://developer.apple.com/documentation/Swift/Hashable)

```

## [ Parameters ](https://developer.apple.com/documentation/swiftui/view/tag\(_:includeoptional:\)#parameters) 

`tag`
    
A [`Hashable`](https://developer.apple.com/documentation/Swift/Hashable) value to use as the view’s tag. 

`includeOptional`
    
If the tag value for `Optional<V>` should also be set.
## [Return Value](https://developer.apple.com/documentation/swiftui/view/tag\(_:includeoptional:\)#return-value)
A view with the specified tag set.
## [Discussion](https://developer.apple.com/documentation/swiftui/view/tag\(_:includeoptional:\)#discussion)
Use this modifier to differentiate among certain selectable views, like the possible values of a [`Picker`](https://developer.apple.com/documentation/swiftui/picker) or the tabs of a [`TabView`](https://developer.apple.com/documentation/swiftui/tabview). Tag values can be of any type that conforms to the [`Hashable`](https://developer.apple.com/documentation/Swift/Hashable) protocol.
This modifier will write the tag value for the type `V`, as well as `Optional<V>` if `includeOptional` is enabled. Containers checking for tags of either type will see the value as set.
In the example below, the [`ForEach`](https://developer.apple.com/documentation/swiftui/foreach) loop in the [`Picker`](https://developer.apple.com/documentation/swiftui/picker) view builder iterates over the `Flavor` enumeration. It extracts the string value of each enumeration element for use in constructing the row label, and uses the enumeration value as input to the `tag(_:)` modifier.
```
struct FlavorPicker: View {
    enum Flavor: String, CaseIterable, Identifiable {
        case chocolate, vanilla, strawberry
        var id: Self { self }
    }


    @State private var selectedFlavor: Flavor? = nil


    var body: some View {
        Picker("Flavor", selection: $selectedFlavor) {
            ForEach(Flavor.allCases) { flavor in
                Text(flavor.rawValue)
                    .tag(flavor)
            }
        }
    }
}

```

The selection type of the [`Picker`](https://developer.apple.com/documentation/swiftui/picker) is an `Optional<Flavor>` and so it will look for tags on its contents of `Optional<Flavor>` type. Since the tag modifier defaults to having `includeOptional` enabled, even though the tag for each option is a non-optional `Flavor`, the tag modifier writes values for both the non-optional, and optional versions of the value, allowing the contents to be selectable by the [`Picker`](https://developer.apple.com/documentation/swiftui/picker).
A [`ForEach`](https://developer.apple.com/documentation/swiftui/foreach) automatically applies a default tag to each enumerated view using the `id` parameter of the corresponding element. If the element’s `id` parameter and the picker’s `selection` input have exactly the same type, or the same type but optional, you can omit the explicit tag modifier.
To see examples that don’t require an explicit tag, see [`Picker`](https://developer.apple.com/documentation/swiftui/picker).
